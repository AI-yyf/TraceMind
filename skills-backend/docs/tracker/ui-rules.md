# UI Rules

- 全站白底，不提供深色模式；时间线唯一强强调色为红色。
- 首页开头固定为英文名 `ArXiv Chronicle`，下一行是中文使命。
- “更新原则”只能出现在页面最末的小字区域。
- 左侧栏只保留活跃主题入口、搜索入口、收藏入口与主题管理入口，不保留大段说明文字。
- 主题页默认组织方式是时间化问题树：以问题阶段为主，论文与分支作为问题阶段内部的内容。
- 论文页必须单栏、圆角、留白充足、文字可连续阅读，不做复杂控制台布局。
- 所有论文封面和正文图片必须使用 `object-contain`，不能裁掉原理图主体。
- 无限下滑依靠渐进渲染与增量显示，不能一次挂载过多长文内容。
- 已成形分支可以进入正文流，但默认折叠；未成熟候选只在阶段末尾或研究视图显示。
- 多条分支重新汇合时，必须插入独立“汇流章节”，不能静默改写主线。
- 主题管理页用于启用、归档、恢复和排序主题，不夹带正文阅读结构。
- 打印样式必须隐藏侧栏、搜索、收藏按钮和其它非正文控件。
