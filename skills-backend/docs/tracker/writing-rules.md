# Writing Rules

- 每篇正式论文正文目标长度为 2000 到 3000 字中文。
- 主体写法是编年体长文，不以分点清单充当正文。
- 全文以“期刊编辑”身份书写，不使用项目自述、开发注释或维护者口吻。
- 文章必须承前启后：
  前文发展到哪里、这篇论文为何在此时出现、它如何用图表和公式解决问题、它仍留下什么问题。
- 文章必须镜像原论文主要章节顺序，不使用通用的“背景/方法/关系”拼贴框架。
- 语言要求严谨、凝练、科学，避免口号化、空泛赞美和弱判断。
- 图表解释必须回到论文问题本身，不能只描述“图里有什么”。
- 每个关键图表都必须有图注或表注，并围绕其在本章论证中的作用展开分析。
- 公式解释必须先说明变量在论文中的角色，再说明它为何重要。
- 行内符号、变量、希腊字母、上下标、算子与块级公式统一用 MathJax 可渲染的 TeX 书写。
- 图注、表注、正文段落与表格单元格中的数学对象也必须使用 TeX。
- 正文应该像一个连贯章节，板块只是叙事停顿，不是项目符号。
- 结尾必须明确指出下一篇论文应接手的未解决问题。
- `problemsOut` 必须补充：
  `problemConstraints`
  `requiredCapabilities`
  `potentialTransferDirections`
- 块级公式只保留真正决定方法机制的核心式子，不堆砌推导。
- 正文图片必须区分 `cover` 与 `evidence figure`；正文图只允许服务问题判断，不能做装饰。
- 表格只保留会改变判断的关键表格，不复刻整页 benchmark。
