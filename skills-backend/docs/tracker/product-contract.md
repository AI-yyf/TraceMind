# Product Contract

`TraceMind` 是一个按主题维护的研究编年站，也是一种书写中的研究之书，而不是论文广场。

- 主题来源是有限预置主题库；当前网页只展示活跃主题集合。
- 主题启用、归档、恢复、排序全部通过网页 UI 完成，`skill` 不参与主题治理。
- 当前正式站点只展示每个主题已经完成深写并发布的论文，以及已成形的分支内容。
- 源头论文采用唯一合法标准：`earliest-representative`。
- 当前阶段默认每个主题只发布 1 篇已审计源头论文；后续每次仅新增 1 篇正式长文。
- 主题页默认展示时间化问题树，而不是单一论文墙。
- 未完成深写的候选只作为研究提示或候选卡出现，不直接进入正式正文流。
- 首页首先像一本书的封面或扉页，而不是规则面板。
- 首页首屏不得出现规则说明、更新原则或维护口径，首屏之后立刻进入纵向时间线入口。
- 论文页始终保持单栏长文，图表、表格、符号与公式作为证据嵌入叙事，而不是做成仪表盘。
- 论文页正文结构固定为：`openingStandfirst -> sections[] -> closingHandoff -> problemsOut`。
- 主题页顶部固定为：主题名、凝练追溯句、期刊式导语，不出现产品化标签墙。
- 研究视图专门展示候选、分支、能力原语与决策日志，不打断正文阅读流。
- 收藏粒度是正文证据块或段落块；导出 PDF 依赖浏览器打印，不引入额外后端。
