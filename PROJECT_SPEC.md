# 溯知集 ArXiv Chronicle - 项目设计规范

## 📌 一、核心定位（不可偏离）

### 1.1 项目定位
- **项目名称**: 溯知集 (ArXiv Chronicle)
- **核心功能**: AI 驱动的学术论文追踪与深度分析系统
- **目标用户**: 学术研究者、研究生、科研团队
- **核心价值**: 帮助用户高效发现、筛选、整合和深入分析某一主题下的多篇学术论文

### 1.2 禁止偏离的方向
- ❌ 禁止做成通用笔记应用
- ❌ 禁止做成社交分享平台
- ❌ 禁止做成论文下载站
- ❌ 禁止做成 AI 聊天机器人
- ❌ 禁止做成翻译工具

## 📌 二、语言模式规范（强制执行）

### 2.1 三种语言模式
| 模式 | 代码 | 主题名 | 关键词 | 内容语言 |
|------|------|--------|--------|----------|
| 简体中文 | `zh` | 仅中文 | 仅中文 | 仅中文 |
| English | `en` | 仅英文 | 仅英文 | 仅英文 |
| 双语模式 | `bilingual` | 中英文都有 | `{中文}/{English}` | 中英文都有 |

### 2.2 语言模式限制
- **选择时机**: 创建主题时选择
- **选择后**: **不可更改**（存入数据库 `Topic.language` 字段）
- **影响范围**: 主题名、关键词、描述、论文内容、阶段名称

### 2.3 内容语言规则
```
如果用户选择「简体中文」模式：
├── 主题名称 → 必须是中文
├── 关键词 → 全部中文
├── 阶段名称 → 中文（问题提出/基础方法/技术改进/应用拓展/综合分析）
├── 论文标题 → 显示中文译名（titleZh）
└── 内容展示 → 全部中文

如果用户选择「English」模式：
├── Topic Name → English only
├── Keywords → English only
├── Stage Names → English (Problem Formulation/Foundation/...)
├── Paper Title → Original English
└── Content Display → English only

如果用户选择「双语模式」：
├── 主题名称 → 中英文都要
├── 关键词 → 每个关键词都是 "{中文}/{English}" 形式
├── 阶段名称 → 中英文都要
├── 论文标题 → 中文译名 + 英文原名
└── 内容展示 → 中英文对照
```

## 📌 四、内容显示规范（强制）

### 4.1 论文显示规则
| 字段 | 简体中文模式 | English 模式 | 双语模式 |
|------|-------------|--------------|----------|
| 标题 | `titleZh` | `title` | `titleZh` + `titleEn` |
| 作者 | 中文显示 | English | 中英对照 |
| 摘要 | `summary` (中文) | `abstract` | 两者都要 |
| 日期 | YYYY年MM月 | YYYY-MM-DD | YYYY年MM月 (YYYY-MM-DD) |

### 4.2 关键词显示规则
```
简体中文: ["大语言模型", "自动驾驶", "端到端"]
English: ["LLM", "Autonomous Driving", "End-to-End"]
双语模式:
  [
    { zh: "大语言模型", en: "LLM" },
    { zh: "自动驾驶", en: "Autonomous Driving" },
    { zh: "端到端", en: "End-to-End" }
  ]
```

### 4.3 阶段名称规范
| 阶段索引 | 简体中文 | English | 双语模式 |
|----------|----------|---------|----------|
| 1 | 问题提出 | Problem Formulation | 问题提出 / Problem Formulation |
| 2 | 基础方法 | Foundation | 基础方法 / Foundation |
| 3 | 技术改进 | Technical Improvement | 技术改进 / Technical Improvement |
| 4 | 应用拓展 | Application | 应用拓展 / Application |
| 5 | 综合分析 | Synthesis | 综合分析 / Synthesis |

| 文字主色 | Gray 900 | `#111827` |
| 文字次色 | Gray 500 | `#6B7280` |

### 5.2 字体规范
- **主字体**: 系统字体栈 `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- **代码字体**: `"SF Mono", Monaco, "Cascadia Code", monospace`
- **标题字号**: 24px (h2), 20px (h3), 16px (h4)

### 5.3 间距规范
- **页面内边距**: 24px
- **卡片内边距**: 16px
- **元素间距**: 8px (紧凑), 16px (标准), 24px (宽松)
- **圆角**: 8px (按钮), 12px (卡片), 16px (模态框)

### 5.4 组件规范
| 组件 | 规范 |
|------|------|
| 按钮主色 | `bg-amber-500 hover:bg-amber-600` |
| 按钮次色 | `bg-gray-100 hover:bg-gray-200` |
| 卡片阴影 | `shadow-sm border border-gray-200` |
| 输入框 | `border border-gray-300 focus:ring-2 focus:ring-amber-500` |

## 📌 七、数据模型规范（强制）

### 7.1 必须的数据库字段
```prisma
model Topic {
  id          String   @id
  nameZh      String              // 简体中文模式必填
  nameEn      String?             // English/双语模式必填
  language    String              // zh / en / bilingual
  summary     String              // 主题描述
  // ... 其他字段
}

model TopicStage {
  name        String              // 主要名称
  nameEn      String?             // English 模式使用
  // ...
}
```

### 7.2 语言字段使用规则
```typescript
// 获取主题显示名称
function getTopicDisplayName(topic: Topic): string {
  if (topic.language === 'zh') return topic.nameZh
  if (topic.language === 'en') return topic.nameEn || topic.nameZh
  if (topic.language === 'bilingual') return `${topic.nameZh} / ${topic.nameEn}`
  return topic.nameZh
}
```

## 📌 九、定时任务规范（强制）

### 9.1 支持的任务类型
| 类型 | 说明 |
|------|------|
| `discover` | 论文发现（渐进式追踪） |
| `refresh` | 数据刷新 |
| `sync` | 数据同步 |

### 9.2 任务配置限制
- Cron 表达式必须有效
- 每个主题最多 3 个定时任务
- 任务失败最多重试 3 次
