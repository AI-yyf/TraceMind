# 溯知理想态蓝图

## 产品定位

溯知不是“搜论文 + 生成摘要”的工具，而是一个研究工作台：

- 它持续发现和筛选论文
- 它围绕主题形成时间化、分支化的节点结构
- 它抽取图、表、公式、章节证据
- 它把节点组织成可阅读、可追问、可导出的研究产物
- 它允许用户通过工作台与正在推进研究的 AI 研究助手协作

## 研究闭环

标准闭环应当是：

1. 发现：多源搜索、聚合、去重、候选入池
2. 筛选：主题配置、阈值、时窗与阶段过滤
3. 抽取：PDF、图片、表格、公式、章节与来源链接
4. 建模：主题时间线、节点 DAG、跨论文关系
5. 判断：节点级结论、开放问题、合并/分叉逻辑
6. 对话：围绕主题、节点、论文、证据的工作台追问
7. 导出：研究摘要、节点文章、主题包与后续行动

## 用户体验原则

- 主题页负责研究结构，不负责所有操作堆叠。
- 节点页和论文页优先是阅读面，而不是数据仪表盘。
- 右侧工作台是研究协作入口，不是附属聊天框。
- 所有时间标签、阶段标签、节点卡片和研究总结都应来自可解释的数据与生成链路。
- 任何“看起来很完整”的内容，都应该能回到论文、节点或证据层验证。

## 当前仓库落点

- 主题、节点、研究页和工作台入口
- 研究调度与 duration-first 任务模型
- 搜索聚合、外部搜索补强与 topic search
- 节点/论文文章流与 evidence surface
- 统一模型配置入口与 Omni gateway
