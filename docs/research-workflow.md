# 研究流程：从线索到判断

溯知的核心不是「生成一段更漂亮的摘要」，而是帮助个人研究者把一次次阅读、搜索、追问和怀疑沉淀成可继续推进的研究结构。

你可以把它理解为一个循环：

```text
问题出现
  -> 发现资料
  -> 筛选论文
  -> 抽取证据
  -> 建立节点
  -> 形成判断
  -> 继续追问
  -> 导出产物
  -> 回到主题记忆
```

每一轮循环都不会覆盖上一轮，而是让主题变得更清楚。

## 第一步：创建一个 Topic

Topic 是溯知里的长期研究对象。它不应该只是一个关键词，而应该接近一个你愿意持续追踪的问题。

好的 Topic 示例：

- 端到端自动驾驶如何处理可解释性和安全边界？
- 具身智能任务规划里，世界模型是否正在取代传统符号规划？
- RAG 系统的评估指标为什么总是和真实用户体验脱节？
- 小模型在端侧 Agent 中的能力边界在哪里？

Topic 的作用是把搜索、论文、节点、对话和导出产物放在同一个上下文里。

## 第二步：发现候选论文

溯知会从 arXiv、OpenAlex、Crossref、Semantic Scholar 等开放学术来源聚合候选论文。这个阶段的目标不是立刻相信所有结果，而是建立一个可以被审视的候选池。

候选论文通常会被分成三类：

- `admitted`：已经进入主题主线，值得持续引用和分析。
- `candidate`：看起来相关，但需要更多证据确认。
- `rejected`：关键词相似或表面相关，但不适合作为当前主题证据。

这种分类能避免研究者被「看起来都相关」的论文淹没。

## 第三步：把论文变成证据对象

一篇论文不只是标题和摘要。溯知更关心论文里可以被追溯的证据：

- PDF 文本和章节结构。
- 图表、表格和图注。
- 公式与方法假设。
- 引用关系和参考文献。
- 支持某个判断的具体片段。

抽取结果不应该被当成绝对真相。它们是研究者检查、追问和修正的材料。

## 第四步：建立研究节点

Node 是 Topic 里的结构单元。它通常对应一个问题、方法、分歧、机制、局限、趋势或阶段性判断。

例如在「端到端自动驾驶」主题下，节点可能是：

- 端到端模型的可解释性问题。
- 多模态感知到规划的接口变化。
- 闭环仿真评估的可信度。
- 大模型是否适合车端实时决策。
- 数据闭环和长尾场景挖掘。

节点让论文不再只是列表，而是进入一个可阅读、可追问、可复用的研究地图。

## 第五步：阅读带证据链的节点文章

溯知的节点文章不是普通摘要。它更像一份研究备忘录：

- 先解释这个节点为什么重要。
- 再分析关键论文和证据。
- 然后综合不同论文之间的关系。
- 最后给出阶段性判断、局限和下一步问题。

理想情况下，用户看到一个判断时，可以继续回到相关论文、图表、公式或引用，而不是只看到一段流畅文字。

## 第六步：在 Workbench 里继续追问

研究不是一次生成结束。你可能会继续问：

- 这个节点里最薄弱的证据是什么？
- 有没有反例论文？
- 哪些结论只是模型推断？
- 如果我要写一段综述，应该引用哪几篇？
- 这个判断在六个月后可能被什么新进展推翻？

Workbench 的价值在于让追问回到当前 Topic、Node 和 Evidence，而不是孤立地发生在一个聊天窗口里。

## 第七步：导出研究产物

当一个主题逐渐稳定后，溯知可以帮助你整理可导出的材料：

- 节点文章。
- 研究简报。
- 论文清单。
- 证据摘录。
- Zotero bundle 或参考文献材料。
- 面向写作、汇报或决策的报告草稿。

导出不是研究的终点，而是把阶段性理解带到外部写作和协作环境里。

## 第八步：保留长期记忆

真正有价值的研究系统，不应该每次都从零开始。溯知希望记录：

- 你已经读过哪些论文。
- 哪些路线被暂时否定。
- 哪些判断仍然不稳定。
- 哪些问题需要下次优先补证据。
- 哪些模型输出曾经被修正。

这就是「个人研究工作台」和「一次性 AI 问答」的根本差异。

## 一个新用户的前 30 分钟

1. 启动前后端，打开前端首页。
2. 在 Settings 中配置一个语言模型和可选视觉模型。
3. 创建一个你真实关心的 Topic。
4. 运行论文发现，先不要急着相信结果。
5. 挑选 3 到 5 篇最相关论文，检查它们的证据对象。
6. 打开节点页，看溯知如何把论文组织成问题结构。
7. 在 Workbench 里问一个反驳性问题，例如「这个节点最可能错在哪里」。
8. 把有价值的回答沉淀回主题判断或下一步阅读计划。

## 研究者仍然是中心

溯知会帮助你整理、抽取、生成和追问，但最终判断仍然属于你。一个好的 AI 研究工作台不应该让人更依赖幻觉，而应该让人更容易看见证据、更容易发现不确定性、更容易回头修正自己。
